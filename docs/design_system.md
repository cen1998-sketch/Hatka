# Техническая документация «Хатка» (v1.1)
## Раздел: Реализация на shadcn/ui

### 1. Визуальный стиль (Theming)
В `tailwind.config.js` необходимо расширить стандартную тему под брендбук «Хатки»:
- **Primary:** Оранжевый (`#FF7F00` — как на кнопке «Найти хатку»).
- **Secondary:** Бирюзовый (`#4DB6AC` — для блока «Подобрали специально для вас»).
- **Radius:** Округление `xl` (примерно 12px-16px), так как интерфейс выглядит мягким и дружелюбным.

### 2. Сопоставление элементов интерфейса с компонентами shadcn/ui

| Элемент интерфейса | Компонент shadcn/ui | Особенности реализации |
| :--- | :--- | :--- |
| Поиск (Даты) | Popover + Calendar | Режим `range` для выбора дат заезда и выезда. |
| Быстрое бронирование | Switch | Оранжевый активный стейт. |
| Типы жилья | Checkbox | Список с `label` для множественного выбора. |
| Слайдер цены | Slider | Двойной ползунок (`range`) для мин/макс цены. |
| Кнопка «Найти» | Button | Размер `lg`, цвет `primary`. |
| Карточка квартиры | Card | Кастомный контент: `CardContent` для фото, `CardFooter` для цены. |
| Рейтинг/Район | Badge | Скругленные плашки с иконками внутри фото. |
| Премиум блок | Carousel | Горизонтальная прокрутка с `CarouselContent` и `CarouselItem`. |
| Вход по Email | Input + Button | Состояние `loading` (spinner) при отправке ссылки. |

### 3. Детальная структура ключевых модулей

#### А. Поисковый фильтр (Sidebar)
Используем вертикальный стек компонентов:
- **Заголовок:** Typography (`h3`).
- **Toggle:** Label + Switch для «Быстрого бронирования».
- **Группа чекбоксов:** Обернуты в `ScrollArea`, если типов жилья станет больше 10.
- **Цена:** Slider с динамическим выводом значений в Input (read-only).
- **Действие:** Button (`variant: default`, `width: full`).

#### Б. Интеллектуальный блок («Специально для вас»)
Реализуется через компонент `Carousel`.
- **Слот рекламы:** Отдельный `CarouselItem` с кастомным фоном (оранжевый градиент) и кнопкой «Получить».
- **Сжатые карточки:** Уменьшенная версия `PropertyCard` с меньшим количеством метаданных (только цена и адрес).

#### В. Шапка профиля (Header)
- **Навигация:** `NavigationMenu` для пунктов «Сдать жилье», «Бронирования».
- **Мессенджер:** Button (`variant: ghost`) с индикатором `Badge` (количество новых сообщений).
- **Профиль:** `Avatar` с выпадающим списком `DropdownMenu` для смены роли (Арендатор <-> Арендодатель).

### 4. Логика состояний (State Management)
Для эффективной работы фронтенда рекомендую:
- **Фильтры:** Хранить в URL-параметрах (через `useQueryState`), чтобы пользователь мог скопировать ссылку с примененными фильтрами.
- **Тип аренды:** Глобальный стейт (Context или Zustand), который переключает набор доступных фильтров в реальном времени.

### 5. Адаптивность (Responsiveness)
- **Desktop:** Левое меню зафиксировано, сетка карточек 2х2 или 3х3.
- **Mobile:** Левое меню прячется в `Drawer` (компонент shadcn), поиск сворачивается в компактную строку сверху.
