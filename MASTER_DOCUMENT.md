# Hatka: Полная техническая спецификация (Master Source of Truth)

Этот документ является единственным авторитетным источником информации по проекту Hatka. Он объединяет требования MVP v1.0, спецификации v1.1 по отелям и актуальные доработки (Magic Link, ЮKassa, iCal).

---

## 1. Обзор проекта
**Hatka** — это премиальная платформа для аренды жилья на срок от одной недели (недельная/долгосрочная аренда).
- **Миссия**: Упростить процесс поиска и сдачи качественного жилья с гарантией оплаты и прозрачностью ролей.
- **Главные отличия**: Строгая модерация, система подписки для арендаторов и отсутствие комиссии для арендодателей (в рамках промо-периода).

## 3. Модели данных (Prisma Schema)
### Основные сущности:
1. **User**: `id`, `email`, `role` (TENANT, LANDLORD, BOTH), `firstName`, `lastName`, `city`, `avatarUrl`, `isDeleted`.
2. **Listing**: `id`, `type` (APARTMENT, HOUSE, ROOM, HOTEL), `status` (DRAFT, PENDING, APPROVED, REJECTED), `details` (JSONB), `pricePerNight`.
3. **Room**: (Для отелей) `id`, `listingId`, `title`, `capacity`, `price`, `quantity`.
4. **Booking**: `id`, `userId`, `listingId`, `id_from_provider`, `status` (WAITING_PAYMENT, HOLD, CAPTURED, CANCELLED).
5. **Referral**: `referrerId`, `referredUserId`, `status` (PENDING, ACTIVATED), `daysEarned`.
6. **Subscription / CommissionFreePeriod**: Срок действия и оставшиеся дни.

## 4. Аутентификация и Роли
### 4.1. Вход через Magic Link
1. Пользователь вводит email.
2. Система проверяет наличие пользователя. Если его нет — создается новый с ролью по умолчанию (выбирается на UI).
3. Генерируется JWT (Access/Refresh) и отправляется ссылка на почту.
4. После клика по ссылке пользователь авторизуется в системе.
*Примечание: Сохранена поддержка входа по паролю для старых аккаунтов, но приоритет — Magic Link.*

### 4.2. Разделение ролей
- **Арендатор (TENANT)**: доступ к поиску, избранному и списку своих бронирований.
- **Арендодатель (LANDLORD)**: доступ к дашборду управления объектами, календарям и финансовой аналитике.
- **Переключение**: В профиле доступна кнопка смены контекста (Role Switcher).

## 5. Создание объявлений (Listings)
### 5.1. Многошаговый Wizard (Квартиры/Дома/Комнаты)
Процесс состоит из **16 подшагов** с обязательным автосохранением (`PATCH /api/listings/:id/step/:step`) каждые 5 секунд или при `blur`.
- **Минимальные требования для публикации**:
    - 3+ фотографии.
    - Описание (300+ символов).
    - Точный адрес (валидация через DaData).
    - Выбранные удобства (Amenities).

### 5.2. Модуль Отелей и Номеров
**Отель** — это зонтичная сущность. Сама недвижимость редактируется как Отель, но бронируются конкретные **Номера (Rooms)**.
- **Редактор отеля**: Одностраничный интерфейс с **якорной навигацией (Sidebar Navigation)**.
- **Поля номера**: Вместимость (основные + доп. места), площадь, этаж, базовые услуги.
- **Лимиты фото номера**: 3–50 штук (JPG, WEBP, HEIC).

## 6. Бронирование и Финансы
### 6.1. Жизненный цикл бронирования
1. **Создание**: Гость выбирает даты. Если "Мгновенное" — сразу оплата, если "По запросу" — Хозяин должен подтвердить в течение 24 часов.
2. **Оплата (ЮKassa)**:
    - **Hold (Преавторизация)**: Деньги замораживаются на карте гостя.
    - **Capture (Списание)**: Деньги переводятся в систему при заезде или подтверждении.
    - **Refund**: Возврат при отмене (согласно правилам отмены).

### 6.2. Поллинг и LIVE-обновления
Дашборд Хозяина опрашивает API каждые 10 секунд (pollingInterval) для моментального отображения новых заявок.

---

## 7. Календарь и iCal
- **Сетка управления**: Единый календарь для всех номеров отеля/объекта. Поддерживает Drag-to-select для массового закрытия дат или изменения цен.
- **Синхронизация (iCal)**:
    - **Экспорт**: Ссылка для Avito/Sutochno/AirBnB.
    - **Импорт**: Cron-задача каждые 15 минут парсит внешние ссылки и блокирует даты в Hatka.

---

## 8. Реферальная система
- **Для Арендатора**: `+4 дня` подписки за приглашенного друга, который зарегистрировался.
- **Для Арендодателя**: `+4 дня` без комиссии за друга, который опубликовал первое одобренное объявление.
- **Интерфейс**: Раздел "Пригласи друга" в ЛК с копированием уникальной ссылки.
