# Hatka: Глобальный мастер-документ (Source of Truth)

Этот документ является финальной технической спецификацией проекта Hatka, объединяющей MVP-документацию и актуальные требования владельца продукта.

---

## 1. Обзор и Технологический Стек
**Hatka** — платформа долгосрочной и недельной аренды (от 7 дней).
- **Frontend**: React 19, Feature-Sliced Design (FSD), RTK Query, React Router 7.
- **Backend**: Node.js + Express.js, Prisma ORM, PostgreSQL 15, Redis (кеш).
- **Интеграции**: ЮKassa (платежи), DaData (адреса), S3 (фото), SendGrid (email).

---

## 2. Пользовательские Роли
- **Арендатор (TENANT / GUEST)**: поиск, бронирование, подписка.
- **Арендодатель (LANDLORD / HOST)**: публикация, управление номерами, календарь, доходы.
- **Оба (BOTH)**: возможность переключения контекста в профиле.

---

## 3. Аутентификация и Безопасность
- **Метод**: Magic Link (вход по email) + JWT (Access 15m / Refresh 7d).
- **Регистрация**: Обязательный выбор роли при первом входе.
- **Безопасность**: Middleware для проверки JWT, Rate limiting, валидация всех данных через Zod.

---

## 4. Жизненный цикл объявления (Listings)
### 4.1. Объекты (Квартиры, Дома, Комнаты)
- **Создание**: `POST /api/listings` создает `DRAFT`.
- **Флоу**: Многошаговый Wizard (до 16 подшагов).
- **Автосохранение**: `PATCH /api/listings/:id/step/:step` (debounce 1.5s).
- **Статусы**: `DRAFT` -> `PENDING` (модерация) -> `APPROVED` (опубликовано) / `REJECTED` (с причиной).

### 4.2. Отели и Номера (Hotel Rooms)
- **Структура**: Объект (Отель) содержит дочерние сущности (Номера).
- **Редактор номеров**: Одностраничный интерфейс с **якорной навигацией** (sidebar). Секции: описание, удобства, фото, настройки бронирования.
- **Лимиты фото**: Минимум 3, максимум 50 на номер.

---

## 5. Бронирование и Календарь
### 5.1. Модуль Booking
- **Типы**: Мгновенное бронирование или "По запросу" (Hold 24ч).
- **Этапы**: `PENDING` -> `CONFIRMED` (Hold -> Capture) -> `ACTIVE` -> `COMPLETED`.
- **LIVE**: Поллинг дашборда (10с) для моментального отображения новых заявок.

### 5.2. Календарь и iCal
- **Интерфейс**: Сетка с поддержкой Drag-to-select для массового изменения цен/статусов.
- **iCal**: Синхронизация (Export/Import) с Avito, Sutochno и др. раз в 15 минут.

---

## 6. Реферальная система
- **Механика**: Ссылка приглашения (+4 дня подписки/без комиссии обоим).
- **Активация**: При регистрации (для гостей) или при первой публикации (для хостов).
